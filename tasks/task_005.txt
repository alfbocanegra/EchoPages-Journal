# Task ID: 5
# Title: Implement Local Storage and Encryption
# Status: done
# Dependencies: 2, 3
# Priority: high
# Description: Enable secure local storage with encryption at rest for all user data.
# Details:
Use SQLite with encryption wrapper for local storage. Implement AES-256 encryption for data at rest using SQLCipher, which provides similar functionality to SQLite Encryption Extension (SEE). Ensure encryption keys are managed securely (Keychain, KeyStore). Encryption should cover journal entries, folders, and folder names. SQLCipher, like SEE, supports encryption of the entire database file, making it appear as white noise to outside observers. The implementation should maintain compatibility with both encrypted and unencrypted databases. SQLCipher uses PBKDF2 for key derivation, which provides enhanced security against brute force, dictionary, and pre-computation attacks. The key management system will use the device-bound key from SecureKeyStorage for all database encryption and rekey operations.

# Test Strategy:
Test data persistence, encryption, and secure key management. Verify data integrity after app restarts. Test encryption of all data types including journal entries, folders, and folder names. Verify that the entire database file is encrypted, including both data and metadata. Test the key management APIs for proper functionality, including initial key setting and key rotation. Verify that encrypted databases appear as white noise to outside observers. Test performance with SQLCipher's key derivation process (PBKDF2) and ensure it meets application requirements. Verify that the device-bound key from SecureKeyStorage is properly used for all database encryption and rekey operations.
