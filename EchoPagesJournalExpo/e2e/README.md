# EchoPages Journal - E2E Testing with Detox

## Overview

This directory contains End-to-End (E2E) tests for the EchoPages Journal application using Detox. E2E tests simulate real user interactions and verify that the complete application works correctly across different scenarios.

## Test Structure

```
e2e/
├── README.md                 # This documentation
├── setup.js                  # Test setup and utilities
├── jest.config.js           # Jest configuration for E2E tests
├── starter.test.js          # Basic starter test (generated by Detox)
└── journal-app.test.js      # Comprehensive journal app tests
```

## Running E2E Tests

### Basic Commands

```bash
# Run E2E tests on iOS simulator (default)
npm run test:e2e

# Run E2E tests on Android emulator
npm run test:e2e:android

# Run E2E tests on iOS simulator
npm run test:e2e:ios

# Build apps for testing
npm run test:e2e:build

# Run all tests (unit + E2E)
npm run test:all
```

### Advanced Commands

```bash
# Build specific platform

# Run with specific configuration
```

## Test Development

### Writing New Tests

1. **Create Test File**: Add new `.test.js` file in `e2e/` directory
2. **Import Utilities**: Use `TestUtils` and `TestConstants` from `setup.js`
3. **Follow Structure**: Use describe/it blocks for organization
4. **Add Test IDs**: Ensure elements have proper test IDs

### Example Test Structure

```javascript
import { TestUtils, TestConstants, MockData } from './setup';

describe('Feature Name', () => {
  beforeEach(async () => {
    // Setup for each test
    await TestUtils.reloadApp();
  });

  it('should perform specific action', async () => {
    // Arrange
    await TestUtils.waitForElement('element-id');
    
    // Act
    await TestUtils.tapElement('button-id');
    await TestUtils.typeText('input-id', 'test text');
    
    // Assert
    await expect(element(by.text('Expected Result'))).toBeVisible();
  });
});
```

### Test Utilities

#### TestUtils
- `waitForElement(testId, timeout)`: Wait for element to be visible
- `tapElement(testId)`: Safely tap an element
- `typeText(testId, text)`: Type text into input
- `clearAndTypeText(testId, text)`: Clear and type text
- `scrollToElement(testId, direction)`: Scroll to element
- `takeScreenshot(name)`: Take screenshot for debugging
- `reloadApp()`: Reload app for clean state

#### TestConstants
- `timeouts`: Predefined timeout values
- `testIds`: Centralized test ID constants
- `text`: Expected text values

#### MockData
- `entries`: Sample journal entries for testing
- `users`: Sample user data for testing

## Best Practices

### Test Organization
1. **Group Related Tests**: Use describe blocks for feature grouping
2. **Clear Test Names**: Use descriptive test names
3. **Independent Tests**: Each test should be independent
4. **Clean Setup**: Use beforeEach for test setup

### Element Selection
1. **Use Test IDs**: Prefer `testID` over text or accessibility labels
2. **Stable Selectors**: Avoid selectors that change frequently
3. **Hierarchical Selection**: Use parent-child relationships when needed

### Error Handling
1. **Wait for Elements**: Always wait for elements before interacting
2. **Timeout Management**: Use appropriate timeouts for different operations
3. **Error Scenarios**: Test both success and failure paths

### Performance
1. **Minimize Screenshots**: Only take screenshots when needed
2. **Efficient Navigation**: Use direct navigation when possible
3. **Clean State**: Reset app state between tests

## Debugging

### Common Issues

#### Element Not Found
```bash
# Check if element exists
await expect(element(by.id('element-id'))).toBeVisible();

# Use longer timeout
await TestUtils.waitForElement('element-id', 10000);
```

#### Test Flakiness
```bash
# Increase timeout
await TestUtils.waitForElement('element-id', 15000);

# Add retry logic
await TestUtils.retry(() => element(by.id('element-id')).tap(), 3);
```

#### Build Issues
```bash
# Clean build
rm -rf ios/build android/app/build
npm run test:e2e:build

# Check Xcode/Android Studio setup
xcodebuild -version
adb devices
```

### Debugging Tools


## Continuous Integration

### GitHub Actions Example

```yaml
name: E2E Tests
on: [push, pull_request]
jobs:
  e2e-tests:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm ci
      - run: npm run test:e2e:build:ios
      - run: npm run test:e2e:ios
      - uses: actions/upload-artifact@v3
        with:
          name: test-results
          path: test-results/
```
